# Open questions: pre-registered, deliberately unanswered

PCR adds structure only when real use demands it — so the questions below are recorded as *open*, not answered. If you (human or agent) hit one in a real project, that is the pain the growth rule waits for: bring the case to the methodology instead of inventing a local answer.

- **Monorepo layout.** One records folder at the root, or one per package? Likely: records follow the agent-instructions files (nesting where those nest) — but no project has needed an answer yet.
- **Concurrent agents.** Two parallel sessions both appending records and map lines — git merges text, but does that hold up in practice?
- **Distill cadence.** "Whenever a human reviews" is deliberately vague. A release ritual? A folder-size trigger? Waiting for a real project's rhythm to show one.
- **The map at 50+ records.** A single-file map may stop scaling; scope-tags are the recorded fallback ([the-map](./the-map.md)).
- **Agent compliance.** Which block rules do agents actually follow, and which wash out mid-session? Observed failures are tuning data for the block — report them to the methodology repo.
